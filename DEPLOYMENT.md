# ☁️ SpendOS Deployment Guide

This document provides step-by-step instructions for deploying SpendOS to production using **Render** (backend), **Vercel** (frontend), and **Neon** (PostgreSQL).

---

## 📋 Prerequisites

| Service | Purpose | Signup |
|---|---|---|
| [Render](https://render.com) | Backend hosting + Redis | Free tier available |
| [Vercel](https://vercel.com) | Frontend hosting (React) | Free tier available |
| [Neon](https://neon.tech) | Serverless PostgreSQL | Free tier available |
| [Groq](https://console.groq.com) | LLM inference (Llama 3.3, Qwen 32B) | Free API key |
| [SerpAPI](https://serpapi.com) | Live web search for vendor discovery | Free tier (100 searches/month) |

## 1️⃣ Database Setup (Neon)

1. Create a new project at [neon.tech](https://neon.tech).
2. Copy the **connection string** from the dashboard. It will look like:
   ```
   postgresql://neondb_owner:<password>@ep-<id>-pooler.<region>.aws.neon.tech/neondb
   ```
3. For SpendOS, prefix it with `+asyncpg` for the async driver:
   ```
   postgresql+asyncpg://neondb_owner:<password>@ep-<id>-pooler.<region>.aws.neon.tech/neondb
   ```

> [!NOTE]
> SpendOS automatically detects Neon URLs and enables SSL. No manual `sslmode` configuration needed.

## 2️⃣ Backend Deployment (Render)

### Option A: Blueprint (Automated)

1. Fork/push the SpendOS repo to GitHub.
2. Go to [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
3. Connect your GitHub repo — Render will auto-detect `render.yaml`.
4. Fill in the required environment variables (see below).
5. Click **Apply**.

### Option B: Manual Setup

1. Create a new **Web Service** on Render.
2. Connect your GitHub repo.
3. Configure:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `cd SpendOS_Backend/smart-procurement && python -m scripts.run_worker & uvicorn app.main:app --host 0.0.0.0 --port $PORT`
4. Add a **Redis** instance (Render has a free tier).

### Environment Variables (Render)

| Variable | Example | Required |
|---|---|---|
| `DATABASE_URL` | `postgresql+asyncpg://...@neon.tech/neondb` | ✅ |
| `REDIS_URL` | Auto-set by Render Redis | ✅ |
| `SECRET_KEY` | Auto-generated by Render | ✅ |
| `GROQ_API_KEY` | `gsk_...` | ✅ |
| `SERP_API_KEY` | `...` | ✅ |
| `ALLOWED_ORIGINS` | `https://your-app.vercel.app` | Optional (has defaults) |

### Run Migrations

After the first deploy, run Alembic migrations via Render's **Shell** tab:
```bash
cd SpendOS_Backend/smart-procurement
alembic upgrade head
```

## 3️⃣ Frontend Deployment (Vercel)

1. Go to [Vercel Dashboard](https://vercel.com/dashboard) → **Add New** → **Project**.
2. Import the SpendOS repo.
3. Configure:
   - **Root Directory**: `SpendOS_Frontend`
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. Add Environment Variable:
   - `VITE_API_BASE_URL` = `https://your-render-service.onrender.com/api`
5. Deploy.

> [!TIP]
> Vercel preview deployments (e.g., `spend-abc123.vercel.app`) are automatically allowed by CORS via the `cors_origins_regex` pattern.

## 🔧 Troubleshooting

| Issue | Solution |
|---|---|
| CORS errors from Vercel preview | Check `ALLOWED_ORIGINS` includes your Vercel domain, or relies on the regex pattern |
| Database SSL errors | Ensure `DATABASE_URL` uses `postgresql+asyncpg://` prefix — SSL is auto-configured for Neon |
| Redis connection failures | Verify `REDIS_URL` is correctly set from Render's Redis service |
| ARQ worker not processing | Check Render logs for worker startup — the `&` in the start command runs it in background |
| Alembic migration failures | Run `alembic upgrade head` manually from Render's Shell tab |
